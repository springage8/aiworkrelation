> 知识蒸馏自《中国的红色政权为什么能够存在？》《井冈山的斗争》《星星之火，可以燎原》

## 一、核心论题

> **"一国之内，在四周白色政权的包围中，有一小块或若干小块红色政权的区域长期地存在，这是世界各国从来没有的事。"**
> ——毛泽东《中国的红色政权为什么能够存在？》（1928年）

AI时代的变革面临同样的结构性问题：**全球AI基础设施几乎100%被垄断资本控制——数据在它们的服务器上，算力在它们的芯片上，模型在它们的参数里。在这种条件下，数据公有制怎么可能从零开始？**

答案是：**建立根据地。**

不是"推翻了旧制度再建立新制度"，而是在旧制度的"缝隙"中先建立新制度的"据点"——让新制度在局部范围内运转起来，证明它的可行性，然后逐步扩大。

本文件的目标：设计数据公有制的**第一个根据地**从哪里建、如何建、如何活、如何扩大。

## 二、为什么需要根据地？

### 2.1 没有根据地的方案都是空谈

07号方案从"全球算法公约"开始——这是从上到下。
09号方案从"国际治理框架"开始——这是从大到小。

两者的共同问题：**没有讲第一步怎么走。**

> 当年在大革命初期，党内也有两种倾向：
> - 一种是"城市中心论"——先占领大城市，再向农村扩展（从苏联学来的）
> - 另一种是"全面开花"——在全国各地同时发动，直接推翻旧政权
> - 实践证明这些都是错误的——正确的道路是"农村包围城市"
>
> 为什么？不是因为农村有什么魔力——而是因为：
> 1. 敌人的力量在大城市最集中——硬碰硬打不过
> 2. 敌人在农村的统治最薄弱——有生存空间
> 3. 农村人口占绝大多数——有群众基础
> 4. 可以在局部地区建立政权——先在小范围内证明新制度的可行性

AI时代同理。

### 2.2 AI时代的"农村"在哪里？

"农村"不是一个地理概念，而是一个**结构性概念**——指的是**敌人力量薄弱、群众需求迫切、可以在局部建立替代性制度的领域**。

| 维度 | AI时代的"城市"（敌人强） | AI时代的"农村"（可突破） |
|------|------------------------|------------------------|
| 行业 | 搜索引擎、社交网络、电商、云计算 | 医疗数据共享、农业数据协作、公共教育数据 |
| 地域 | 硅谷、北京、深圳 | 全球南方的二线城市、欧洲的中小国家 |
| 数据 | 用户行为数据（已被平台锁定） | 公共数据（政府数据、科研数据、气象数据） |
| 应用 | 聊天机器人、内容推荐 | 公共服务（交通、环境、健康监测） |

## 三、井冈山在哪里？

### 3.1 选择标准

根据我当年选择井冈山的经验（罗霄山脉中段），一个好的根据地必须具备五个条件：

```
① 敌人统治的薄弱环节
   → 垄断资本没有深度渗透的领域
   → 或者已有替代方案生存空间的领域

② 良好的群众基础
   → 存在迫切的社会需求，现有的资本方案无法满足
   → 群众有参与变革的意愿

③ 可持续的经济模式
   → 能自我维持、自我发展
   → 不依赖外部资助（或早期有限依赖）

④ "地形"优势
   → 制度上的保护空间（法律空白区、地方政府支持）
   → 技术上的可防卫性（开源架构、分布式部署）

⑤ 可扩展性
   → 模式可复制、可推广
   → 不是一次性实验
```

### 3.2 三个候选根据地

根据上述条件的综合评估，三个最可行的方向：

#### 候选一：拉美国家的公共卫生数据信托

**地点：** 智利或巴西

**为什么是这里：**
- 拉美国家有强烈的"数字主权"政治意愿——智利正在起草数据保护法，巴西有《通用数据保护法》（LGPD）
- 公共卫生是"低敏感、高需求"领域——没有人反对"用AI诊断疾病"
- 中国/华为在拉美有基础设施支持——海底光缆、数据中心（但不依赖华为——必须是本地主导的项目）
- 人口规模适中（智利1900万），适合做试点

**具体做什么：**
```
建立"国家公共卫生数据公共信托"：

数据来源：
  → 公立医院的匿名化诊疗数据
  → 公共卫生监测数据（传染病、慢性病）
  → 居民自愿贡献的健康行为数据

数据用途：
  → 训练本地化的AI诊断模型（针对拉美常见病——登革热、恰加斯病、结核病）
  → 优化公共卫生资源配置
  → 制药研发（不付费使用数据的新药，其定价受公共信托约束）

治理结构：
  → 理事会：政府1/4 + 医疗机构代表1/4 + 公民代表1/4 + 技术专家1/4
  → 收益分配：AI产生的经济效益（如更精准的预防减少医疗支出）→ 50%回馈公共卫生系统，30%给数据贡献者（数据红利），20%运营成本

成功指标：
  → 2年内覆盖50%以上的公立医院数据
  → 至少1个AI诊断模型投入实际使用（准确率>90%）
  → 居民自愿参与率>30%
  → 证明数据公有制比私有制更高效——共享数据训练的模型比单个医院训练的模型准确率高X%
```

**风险与应对：**
| 风险 | 应对 |
|------|------|
| 美国施压（"数据公有化是没收"） | 强调这是"公共信托"，不是"没收"——类似公共图书馆 |
| 政治换届导致政策中断 | 写入法律而非行政命令；争取跨党派支持 |
| 技术能力不足 | 与本地大学合作；利用开源AI工具 |
| 数据质量差 | 分阶段——先用高质量的三甲医院数据，再覆盖基层 |

#### 候选二：欧洲城市的交通数据合作社

**地点：** 德国某个城市（如汉堡）或荷兰某个城市（如乌得勒支）

**为什么是这里：**
- EU有全球最强的数据保护立法（GDPR），且正在制定《数据治理法案》（DGA）——法律环境最友好
- 欧洲城市有强大的市政管理传统——交通是市政府直接控制的服务
- 欧洲有合作社法律框架——数据合作社有明确的法律地位

**具体做什么：**
```
建立"城市交通数据合作社"：

数据来源：
  → 市民自愿贡献的GPS出行数据（通过App）
  → 公共交通系统的运营数据
  → 市政传感器的交通流量数据

数据用途：
  → 优化公共交通路线和班次
  → 减少拥堵（AI调度红绿灯）
  → 为城市规划提供决策依据

治理结构：
  → 合作社成员（数据贡献者）一人一票
  → 选举管理委员会
  → 任何使用数据的外部实体（商业公司）必须付费，收益归合作社成员

与资本主义方案的区别：
  → Uber/Waze的方案：采集用户数据→优化导航→卖广告/卖数据→利润归股东
  → 合作社方案：采集市民数据→优化交通→改善公共出行→收益归市民
  → 技术几乎一样，但所有权和分配完全不同
```

**注意：** 欧洲适合做试点，但欧洲的"改良主义"传统可能导致它止步于合作社——而不进一步挑战所有制。在拉美做试点更具变革潜力——因为拉美的不平等更极端，变革的驱动力更强。

#### 候选三：中国某个城市的"外卖骑手数据合作社"

**地点：** 中国某个二线城市（如长沙、成都）

**为什么是这里：**
- 中国有全球规模最大的零工劳动者群体——仅美团就有700万+骑手
- 中国的数字基础设施最发达——但数据全部归平台
- 骑手的被剥削程度全球最高——平台抽成高达20-30%，无劳动保障
- 中国有工会法律框架——虽然目前没有独立工会，但"集体协商"在法律层面是允许的

**具体做什么：**
```
建立"骑手数据合作社"（从最简单的开始）：

第一步：共同监控
  → 骑手共同记录和分析平台的算法规则变化
  → 建立一个"算法规则追踪数据库"——平台改规则，大家一起分析
  → 这不是"技术对抗"，是"信息对称化"

第二步：集体议价
  → 当骑手有了平台算法规则的数据，就有了议价能力
  → "如果单价降到X以下，全体下线15分钟"
  → 数据让集体行动成为可能

第三步：自有平台
  → 骑手合作社推出自己的派单App
  → 初期可以从"非高峰时段"开始——平台不愿接的单（深夜配送、远距离配送）
  → 慢慢扩大
  → 合作社不抽成或抽成5%（用于运营成本），其余全归骑手

成功指标：
  → 12个月内覆盖1000+骑手
  → 合作社平台配送效率不低于商业平台
  → 骑手收入提高20%+（仅靠降低抽成）
  → 数据所有权真正掌握在骑手手中
```

**风险提示：**
- 中国的政治环境对独立组织最严格——这是一个高风险选项
- 但在中国成功的示范效应也最大——因为中国是数字零工经济的全球中心
- **策略：不能直接对抗现有法律框架，要在"劳动者权益"和"数据安全法"的框架内运作**

---

### 3.3 我的推荐

综合评估，**第一个根据地最应该建立在拉美（候选一）**。

理由：
1. 政治空间最大——拉美国家有数字主权的政治意愿
2. 国际影响力强——成功后会带动其他全球南方国家
3. 风险可控——不会直接对抗美国或中国的核心利益
4. 有可用的基础设施——华为等中国企业在拉美有合作基础
5. 示范效应好——从拉美到非洲到亚洲，有可复制性

## 四、根据地的经济模式

一个根据地能否存活，关键在于它**能不能自己养活自己**。

### 4.1 收入来源

数据公共信托在经济上必须是自持的——不需要外部资助就能运转。

```
早期（1-2年）：
  启动资金：来自公共资金（政府拨款、research grant）或公益基金
  规模小，成本低——一个城市的数据信托，服务器+人员，年运营成本约50-200万美元

中期（2-5年）：
  主要收入：数据使用许可费
  保险公司（用健康数据优化保险产品）→ 付费
  制药公司（用医学数据辅助新药研发）→ 付费
  城市规划部门（用交通数据优化城市管理）→ 付费（政府内部预算）
  → 对商业用户的收费足够覆盖运营成本

成熟期（5年+）：
  数据红利分配——除了覆盖运营成本，还有盈余分配给数据贡献者
  这证明"数据公有制是可持续的"
```

### 4.2 分配机制

```
收入分配公式：

年度总收入 = Σ(商业用户数据许可费) + 公共预算支持

分配优先级：
  第一：运营成本（服务器、带宽、人员工资）——约30%
  第二：公共储备基金（技术升级、风险应对）——约20%
  第三：数据红利（分配给数据贡献者）——约50%

个人数据红利分配：
  基础份额（60%）：等额分配给所有数据贡献者
  ——因为数据的本质是社会的，不是个人的
  ——每个人的数据单独价值有限，集体价值巨大
  贡献份额（40%）：按数据量/质量浮动
  ——必要的激励机制——否则没有人愿意贡献更多数据
```

## 五、根据地的组织模式

### 5.1 治理原则

```
民主集中制在数据公有制的应用：

民主：
  → 数据贡献者（即"数据公民"）一人一票选举理事会
  → 重大决策（数据许可、收益分配比例、规则变更）必须经全体投票
  → 算法透明——所有数据使用记录对成员可见

集中：
  → 理事会负责日常管理决策
  → 技术团队有专业自主权（不是"每行代码都要投票"）
  → 一旦做出决策，必须执行——这是效率的保障

关键：民主和集中的平衡
  → 没有民主 = 新形式的"数据官僚主义"
  → 没有集中 = 无效率、无法做任何决策
```


## 六、从根据地到全国——扩大路径

### 6.1 横向扩展：从1个到10个

当一个数据信托试点成功运行2-3年后，开始复制：

```
复制策略：

第一步：文档化——把试点的所有流程、技术方案、法律文件做成"开源工具包"
  让下一个城市不需要从零开始
  降低复制成本

第二步：培训——第一批试点的运营者成为"种子教官"
  去下一个城市帮助建立新试点
  经验是最好的教材

第三步：网络化——各个试点之间建立数据共享协议
  数据信托之间互相连接——数据在更大范围内流动，价值更大
  形成"数据信托联盟"——共同议价、共同标准、共同治理

第四步：标准化——在联盟基础上制定"数据信托国家标准"
  从"各自探索"到"统一规范"
  为全国层面的立法提供实践依据
```

### 6.2 纵向深化：从低敏感到高敏感

```
扩展领域：

第一阶段：公共数据（政府开放数据、气象数据、科研数据）
  → 最容易——政府有开放数据的动力
  → 数据质量高、争议少
  → 建立信任和技术基础

第二阶段：民生数据（健康数据、交通数据、教育数据）
  → 与民众日常生活密切相关
  → 成功的获得感最强
  → 一个疾病早筛模型救人性命 → 比一百篇文章都有效

第三阶段：经济数据（消费数据、供应链数据、金融数据）
  → 触动核心利益——这才是真正的"生产资料"
  → 到这个阶段，资本已经开始反扑
  → 此时需要政治力量而不是技术力量来推进

第四阶段：算法管理权
  → 不只是数据公有，还有"算法权力"的民主化
  → 从"你的数据归你管"到"你的劳动过程归你管"
  → 这是最彻底的变革
```

### 6.3 从经济到政治

根据地不仅仅是"经济实验"，它同时也是"政治实体"。

```
数据信托的政治意义：

1. 它是新的社会关系的微观模型
  → 在这里，数据不是商品，是公共财富
  → 在这里，算法不是黑箱，是透明工具
  → 在这里，劳动者不是工具，是决策者
  → 它展示了"另一种AI社会"是可行的

2. 它是锻炼干部的学校
  → 数据信托的运营需要技术、治理、法律、组织能力
  → 这些能力只有在实践中才能培养
  → 每一次成功的运营，都是在培养未来的变革领导者

3. 它是力量的积累点
  → 数据信托不只是管理数据，也在管理"人"
  → 数据贡献者之间的联系 → 可能发展为政治组织
  → 数据信托可以是"工会+合作社+公共事业"的混合体
```

## 七、根据地要避免的错误

### 7.1 历史上的教训

**教训一：1927年城市暴动的失败**
- 在上海、长沙等大城市发动工人武装暴动
- 全部失败——因为敌人在城市的统治太强了
- 教训：**在敌人力量最强的地方发起挑战，不是勇敢，是鲁莽**

**对应到AI时代：**
- 不要在一开始就和Google、OpenAI正面竞争
- 不要试图"推翻AI资本主义"
- 要在敌人不重视的地方（公共数据、卫生数据、农业数据）建立根据地

**教训二：王明"左"倾冒险主义**
- 要求红军攻打大城市，和国民党主力决战
- 结果：第五次反围剿失败，被迫长征
- 教训：**在力量对比不利时，不要进行阵地战——要打游击战**

**对应到AI时代：**
- 不要期待"一个方案就能改变整个AI行业"
- 不要高估短期内制度变革的可能性
- 要打"游击战"——在局部地区取得小胜利，积累力量

**教训三：张国焘的分裂主义**
- 在长征途中，不服从中央指挥，另立中央
- 结果：分裂削弱了革命力量
- 教训：**整体战略的统一是必要的——分散行动不等于各自为政**

**对应到AI时代：**
- 各个根据地的数据信托应该遵循共同的原则和标准
- 不是"各自发明一套数据公有制"
- 要有一个统一的理论框架（这个仓库提供的）和实践指南

### 7.2 当前要避免的错误

```
错误一：等待完美方案
  "理论还不够完善，等我们把所有细节都想清楚了再动手"
  → 永远不会有完美的方案
  → 实践是检验真理的唯一标准——先干起来，在干中改进

错误二：急于扩大
  "这个试点不错，我们马上在全国推广"
  → 欲速则不达——没有稳固的基础就扩大，只会崩溃
  → 先在一个地方扎深根，再复制

错误三：闭门造车
  "我们自己设计、自己建设、自己运营"
  → 不和当地群众结合的组织注定失败
  → 数据信托必须从一开始就是"群众自己的项目"

错误四：技术决定论
  "技术方案好了，自然有人用"
  → 公共信托的核心不是技术（虽然技术重要）
  → 核心是信任、治理、分配——这些是社会问题，不是技术问题

错误五：悲观主义
  "这点规模有什么用？改变不了大局。"
  → 星星之火，可以燎原
  → 每一个成功的试点都是对"别无选择"论的最好反驳
```

## 八、总结：从井冈山到全中国

1927年，湘赣边界的秋收起义失败了。毛泽东带着残余部队上了井冈山。

当时没有人能想到，这个偏远的山区根据地，最终会发展成为中国革命胜利的起点。

AI时代同样如此。今天在一个拉美城市建立的公共卫生数据信托——规模小、影响有限、随时可能被关停——但它代表着一种可能性：**在AI垄断资本之外的另一种选择。**

> **"马克思主义者不是算命先生，未来的发展和变化，只应该也只能说出个大的方向，不应该也不可能机械地规定时日。"**
> ——毛泽东《星星之火，可以燎原》

但大的方向是清楚的：

```
一个数据信托 → 十个数据信托 → 一百个数据信托
一个城市 → 一个国家 → 一个大陆
公共卫生 → 民生数据 → 经济数据
经济实验 → 政治力量 → 制度变革
```

> **"中国是全国都布满了干柴，很快就会燃成烈火。"**
>
> AI时代同样是全国——不，全球——都布满了干柴。
> AI替代失业、数据剥削、算法异化——这些矛盾正在积累。
> 我们所缺的，不是"条件"，而是一点**星火**。

> **"我所说的中国革命高潮快要到来，决不是如有些人所谓'有到来之可能'那样完全没有行动意义的、可望而不可即的一种空的东西。它是站在海岸遥望海中已经看得见桅杆尖头了的一只航船，它是立于高山之巅远看东方已见光芒四射喷薄欲出的一轮朝日，它是躁动于母腹中的快要成熟了的一个婴儿。"**
>
> AI时代的新社会，正在旧社会的母腹中躁动。

## 附录：根据地建设的检查清单

### 准备阶段
- [ ] 是否选对了地点？（敌人薄弱+群众需求+政治空间）
- [ ] 是否有至少3个全职组织者？
- [ ] 是否完成了对当地数据的"调查研究"？
- [ ] 是否获得了至少一个地方政府的非正式支持？
- [ ] 是否有了实验期的运营资金（不少于6个月）？

### 启动阶段
- [ ] 是否召开了第一次数据公民大会（参与人数>100）？
- [ ] 是否选举产生了理事会？
- [ ] 是否发布了数据信托章程（中英本地语言三语）？
- [ ] 是否部署了开源数据管理平台？
- [ ] 是否签署了第一个数据使用协议？

### 运营阶段
- [ ] 是否有明确的收益分配机制并运行透明？
- [ ] 是否有独立的算法监督委员会？
- [ ] 数据贡献者数量是否持续增长（月增>10%）？
- [ ] 是否有至少1个实际产出的AI应用？
- [ ] 覆盖群体的满意度是否>70%（定期调查）？

### 扩展阶段
- [ ] 是否完成了"工具包"文档化？
- [ ] 是否有至少2个"种子教官"可输出到新试点？
- [ ] 是否与至少1个其他试点建立了联盟关系？
- [ ] 是否推动了至少1项有利于数据公有的立法提案？
- [ ] 是否有媒体关注（至少1篇主流媒体深度报道）？
