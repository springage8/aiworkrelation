## I. 핵심 명제

> **"백색 정권에 포위된 가운데 하나 또는 소수의 작은 붉은 정권이 장기적으로 존재하는 것 — 이것은 세계 어디에서도 전례가 없는 일이다."**
> — 모택동, *「중국의 붉은 정권은 왜 존재할 수 있는가?」*(1928)

AI 시대의 전환은 동일한 구조적 문제에 직면한다: **전 세계 AI 인프라의 거의 100%가 독점 자본에 의해 통제되고 있다 — 데이터는 그들의 서버에, 칩은 그들의 데이터센터에, 모델은 그들의 파라미터에 있다. 데이터 공동 소유를 어떻게 무에서 시작할 것인가?**

정답: **근거지를 건설하라.**

"먼저 낡은 체제를 타도한 다음 새로운 것을 건설하는 것"이 아니라, 낡은 체제의 **틈새** 속에서 새로운 체제의 "거점"을 건설하는 것이다. 새로운 체제가 지역적으로 작동하게 하고, 그 실현 가능성을 입증한 다음, 확장하는 것이다.

이 문서의 목표: 데이터 공동 소유를 위한 **첫 번째 근거지**를 설계하는 것 — 어디에 건설할지, 어떻게 건설할지, 어떻게 유지할지, 어떻게 확장할지.

## II. 왜 근거지인가?

### 2.1 근거지 없는 제안은 공론(空論)이다

문서 07은 "글로벌 알고리즘 협약"에서 시작한다 — 하향식(top-down)이다.
문서 09는 "국제 거버넌스 프레임워크"에서 시작한다 — 큰 것에서 작은 것으로(big-to-small) 간다.

공통된 문제: **어느 것도 첫걸음을 설명하지 못한다.**

> 중국 혁명 초기에는 두 가지 경향이 있었다:
> - "도시 중심" — 먼저 대도시를 점령한 다음 농촌으로 확장 (소련을 모방)
> - "동시 봉기" — 모든 곳에서 동시에 일어나 낡은 체제를 직접 타도
> - 실천은 둘 다 틀렸음을 증명했다 — 올바른 길은 "농촌이 도시를 포위하는 것"이었다
>
> 왜? 농촌에 마법이 있었기 때문이 아니라, 다음과 같은 이유 때문이다:
> 1. 적의 세력은 대도시에 집중되어 있었다 — 정면 대결은 승산이 없었다
> 2. 적의 통치는 농촌에서 가장 취약했다 — 생존할 공간이 있었다
> 3. 농촌 인구가 압도적 다수였다 — 대중적 기반
> 4. 새로운 정치 권력을 지역적으로 수립할 수 있었다 — 작은 지역에서 먼저 새로운 체제를 증명

AI 시대에도 동일한 논리가 적용된다.

### 2.2 AI 시대의 "농촌"은 어디인가?

"농촌"은 지리적 개념이 아니라 **구조적 개념**이다: **적의 세력이 약하고, 대중의 필요가 절박하며, 대안적 제도를 지역적으로 수립할 수 있는** 영역이다.

| 차원 | AI 시대의 "도시" (적이 강함) | AI 시대의 "농촌" (돌파 가능) |
|-----------|------------------------------|----------------------------------------------|
| 산업 | 검색, 소셜미디어, 전자상거래, 클라우드 컴퓨팅 | 공공보건 데이터 공유, 농업 데이터 협력, 공공교육 데이터 |
| 지역 | 실리콘밸리, 베이징, 선전 | 글로벌 사우스 2차 도시, 유럽의 소국가들 |
| 데이터 | 사용자 행동 데이터 (플랫폼에 잠김) | 공공 데이터 (정부, 연구, 기상) |
| 응용 | 챗봇, 콘텐츠 추천 | 공공 서비스 (교통, 환경, 건강 모니터링) |

## III. 징강산은 어디인가?

### 3.1 선정 기준

징강산(나샤오산맥) 선정 경험에 기초할 때, 좋은 근거지는 다섯 가지 조건을 충족해야 한다:

```
1. 적 통치의 취약 고리
   → 독점 자본이 깊이 침투하지 않은 지역
   → 또는 대안적 해결책이 이미 생존할 수 있는 지역

2. 좋은 대중 기반
   → 기존 자본의 해결책으로는 충족할 수 없는 절박한 사회적 필요
   → 변화에 참여하려는 대중의 의지

3. 지속 가능한 경제 모델
   → 스스로 유지되고 발전할 수 있어야 함
   → 외부 자금에 의존하지 않음 (또는 제한된 초기 의존만)

4. "지형" 이점
   → 제도적 보호 공간 (법적 사각지대, 지방정부 지원)
   → 기술적 방어 가능성 (오픈소스 아키텍처, 분산 배포)

5. 확장성
   → 모델이 복제되고 확산될 수 있어야 함
   → 일회성 실험이 아니어야 함
```

### 3.2 세 가지 근거지 후보

#### 후보 1: 라틴아메리카 공공보건 데이터 트러스트

**위치:** 칠레 또는 브라질

**선정 이유:**
- 라틴아메리카 국가들은 강한 "디지털 주권" 정치 의지를 가지고 있음 — 칠레는 데이터 보호법을 준비 중, 브라질은 LGPD 보유
- 공공보건은 "민감도가 낮고 필요성이 높은" 분야 — "AI로 질병을 진단하자"는 누구도 반대하지 않음
- 라틴아메리카에 중국/화웨이 인프라 존재 — 해저 케이블, 데이터센터 (단, 지역 주도여야 함)
- 적절한 인구 규모 (칠레 1900만 명) — 파일럯 프로젝트에 적합

**구체적 계획:**
```
"국립 공공보건 데이터 공공 트러스트" 설립:

데이터 출처:
  → 공공병원의 익명화된 진료 데이터
  → 공중보건 감시 데이터 (전염병, 만성질환)
  → 주민들이 자발적으로 제공하는 건강 행동 데이터

데이터 용도:
  → 지역화된 AI 진단 모델 훈련 (라틴아메리카 흔한 질병 — 뎅기열, 샤가스병, 결핵)
  → 공공보건 자원 배분 최적화
  → 제약 R&D (데이터를 사용한 신약은 가격이 트러스트에 의해 제한되어야 함)

거버넌스 구조:
  → 위원회: 정부 1/4 + 의료기관 대표 1/4 + 시민 대표 1/4 + 기술 전문가 1/4
  → 수익 분배: AI 창출 경제적 이익 → 50% 공공보건 시스템 환원, 30% 데이터 기여자 (데이터 배당), 20% 운영비

성공 지표:
  → 2년 이내: 공공병원 데이터의 50% 이상 커버
  → 최소 1개의 AI 진단 모델 배포 (정확도 >90%)
  → 자발적 주민 참여율 >30%
  → 데이터 공동 소유가 더 효율적임을 입증 — 공유 데이터로 훈련된 모델이 단일 병원 모델보다 X% 더 정확함
```

**위험:**
| 위험 | 대응 |
|------|----------|
| 미국의 압력 ("데이터 공유화는 몰수다") | "공공 트러스트"이지 "몰수"가 아님을 강조 — 공공도서관과 유사 |
| 정권 교체로 정책 중단 | 행정명령이 아닌 법률로 규정; 초당적 지지 확보 |
| 기술 역량 부족 | 현지 대학과 협력; 오픈소스 AI 도구 사용 |
| 데이터 품질 낮음 | 단계적 접근 — 고품질 3차 병원 데이터부터 시작, 이후 확장 |

#### 후보 2: 유럽 도시 교통 데이터 협동조합

**위치:** 독일 도시 (예: 함부르크) 또는 네덜란드 도시 (예: 위트레흐트)

**선정 이유:**
- EU는 세계에서 가장 강력한 데이터 보호 법안(GDPR)을 보유하고 있으며 데이터 거버넌스 법안을 준비 중 — 가장 유리한 법적 환경
- 유럽 도시들은 강력한 시정부 전통을 보유 — 교통은 시 정부가 직접 통제
- 유럽은 협동조합 법적 프레임워크 보유 — 데이터 협동조합이 명확한 법적 지위를 가짐

**구체적 계획:**
```
"도시 교통 데이터 협동조합" 설립:

데이터 출처:
  → 시민의 자발적 GPS 이동 데이터 (앱을 통해)
  → 대중교통 시스템 운영 데이터
  → 도시 센서 교통 데이터

데이터 용도:
  → 대중교통 노선 및 시간표 최적화
  → 교통 혼잡 감소 (AI 교통 신호등 제어)
  → 도시 계획을 위한 데이터 제공

거버넌스 구조:
  → 협동조합원 (데이터 기여자), 1인 1표
  → 선출된 관리 위원회
  → 데이터를 사용하는 모든 외부 기관(상업 회사)은 비용을 지불해야 하며, 수익은 협동조합원 소유

자본주의 해결책과의 차이:
  → 우버/웨이즈: 데이터 수집 → 내비게이션 최적화 → 광고 판매/데이터 판매 → 주주에게 이익
  → 협동조합: 시민 데이터 수집 → 교통 최적화 → 공공 서비스 개선 → 시민에게 혜택
  → 거의 동일한 기술, 완전히 다른 소유권과 분배
```

#### 후보 3: 중국 도시 "배달 라이더 데이터 협동조합"

**위치:** 중국 2선 도시 (예: 창사, 청두)

**선정 이유:**
- 중국은 세계 최대의 긱 노동자 인구 보유 — 메이투안(Meituan)만 700만 명 이상의 배달 라이더
- 중국의 디지털 인프라는 가장 발전됨 — 그러나 모든 데이터는 플랫폼에 귀속
- 라이더 착취는 세계에서 가장 극심한 수준 — 플랫폼 수수료 최대 20-30%, 노동 보호 없음
- 중국에는 노동조합 법적 프레임워크가 존재 (현재 독립 노조는 없지만), "단체 교섭"은 법적으로 허용됨

**구체적 계획:**
```
"라이더 데이터 협동조합" 설립 (가장 단순하게 시작):

1단계: 공동 모니터링
  → 라이더들이 플랫폼 알고리즘 규칙 변경 사항을 공동으로 기록하고 분석
  → "알고리즘 규칙 추적 데이터베이스" 구축 — 플랫폼이 규칙을 변경하면 모두 함께 분석
  → 이것은 "기술적 대결"이 아니라 "정보의 대칭성" 확보

2단계: 단체 교섭
  → 라이더들이 플랫폼 알고리즘 규칙 데이터를 확보하면 교섭력을 얻음
  → "건당 배달료가 X 이하로 떨어지면 모두 15분간 오프라인"
  → 데이터가 집단 행동을 가능하게 함

3단계: 자체 플랫폼
  → 라이더 협동조합이 자체 배차 앱 출시
  → 초기: "비성수기 시간대"부터 시작 — 플랫폼이 원하지 않는 주문 (심야 배달, 장거리)
  → 점진적으로 확장
  → 협동조합 수수료 5% (운영비), 나머지는 라이더에게

성공 지표:
  → 12개월 이내: 1000명 이상의 라이더 커버
  → 협동조합 플랫폼 배차 효율이 상업 플랫폼에 뒤지지 않음
  → 라이더 소득 20% 이상 증가 (수수료 감소만으로)
  → 데이터 소유권이 진정으로 라이더의 손에
```

### 3.3 권장 사항

종합 평가: **첫 번째 근거지는 라틴아메리카(후보 1)여야 한다.**

이유:
1. 가장 큰 정치적 공간 — 라틴아메리카 국가들은 디지털 주권에 대한 정치적 의지를 가지고 있음
2. 강력한 국제적 영향력 — 성공 시 다른 글로벌 사우스 국가들에 영감을 줄 것
3. 통제 가능한 위험 — 미국이나 중국의 핵심 이해관계와 직접 충돌하지 않음
4. 활용 가능한 인프라 — 중국 기업들이 라틴아메리카에 협력 기반을 보유
5. 강력한 시연 효과 — 라틴아메리카에서 아프리카, 아시아로 복제 가능

## IV. 근거지 경제 모델

근거지는 **경제적으로 자립할 수 있을 때**만 생존한다.

### 4.1 수익원

데이터 트러스트는 경제적으로 자립해야 한다 — 외부 보조금이 필요하지 않아야 한다.

```
초기 단계 (1-2년):
  창업 자금: 공공 자금 (정부 보조금, 연구 보조금) 또는 자선 재단
  소규모, 저비용 — 도시 수준 데이터 트러스트, 서버 + 직원, 연간 약 50-200만 달러

중기 단계 (2-5년):
  주요 수익: 데이터 사용 라이선스 수수료
  보험사 (건강 데이터 활용 제품 최적화) → 수수료 지불
  제약회사 (의료 데이터 R&D 활용) → 수수료 지불
  도시 계획 부서 (교통 데이터 활용) → 수수료 지불 (정부 예산)
  → 상업 사용자 수수료로 운영비 충당 가능

성숙 단계 (5년 이상):
  데이터 배당 분배 — 운영비 초과 잉여분을 데이터 기여자에게 분배
  이는 "데이터 공동 소유가 지속 가능함"을 증명
```

### 4.2 분배 메커니즘

```
수익 분배 공식:

연간 총 수익 = ∑(상업 사용자 데이터 라이선스 수수료) + 공공 예산 지원

분배 우선순위:
  첫째: 운영비 (서버, 대역폭, 직원) — 약 30%
  둘째: 공공 준비금 (기술 업그레이드, 위험 대응) — 약 20%
  셋째: 데이터 배당 (데이터 기여자에게 분배) — 약 50%

개인 데이터 배당 분배:
  기본 지분 (60%): 모든 데이터 기여자에게 동등하게 분배
    — 데이터의 본질은 사회적이지 개인적이 아니기 때문
    — 개별 데이터 가치만으로는 미미하지만, 집단적 가치는 엄청남
  기여 지분 (40%): 데이터 양/질에 따라 변동
    — 필요한 인센티브 — 그렇지 않으면 아무도 더 많은 데이터를 기여하지 않음
```

## V. 근거지 조직 모델

### 5.1 거버넌스 원칙

```
데이터 공동 소유에 적용된 민주집중제:

민주주의:
  → 데이터 기여자 ("데이터 시민")가 1인 1표로 위원회 선출
  → 주요 결정 (데이터 라이선싱, 분배 비율, 규칙 변경)은 총투표에 회부
  → 알고리즘 투명성 — 모든 데이터 사용 기록을 구성원이 열람 가능

집중:
  → 위원회가 일상적 관리 결정을 담당
  → 기술팀은 전문적 자율권 보유 ("모든 코드 한 줄을 투표해야 하는 것은 아님")
  → 결정이 내려지면 반드시 실행 — 이것이 효율성을 보장

핵심: 민주주의와 집중의 균형
  → 민주주의 없음 = 새로운 형태의 "데이터 관료제"
  → 집중 없음 = 비효율, 어떤 결정도 내릴 수 없음
```

## VI. 근거지에서 전국으로 — 확장 경로

### 6.1 수평적 확장: 1에서 10으로

하나의 데이터 트러스트 파일럯이 2-3년간 성공적으로 운영된 후, 복제를 시작한다:

```
복제 전략:

1단계: 문서화 — 모든 파일럯 프로세스, 기술 솔루션, 법적 문서를 "오픈소스 툴킷"으로 전환
  다음 도시는 처음부터 시작할 필요 없음
  복제 비용 절감

2단계: 훈련 — 1차 파일럯 운영자가 "씨앗 교관"이 됨
  다음 도시로 가서 새로운 파일럯 구축 지원
  경험이 최고의 교과서

3단계: 네트워킹 — 트러스트 간 데이터 공유 협정 체결
  데이터 트러스트가 연결됨 — 더 넓은 영역에서 데이터 흐름, 더 큰 가치
  "데이터 트러스트 연합" 형성 — 공동 교섭, 표준, 거버넌스

4단계: 표준화 — 연합 경험에 기반한 "국가 데이터 트러스트 표준" 개발
  "각자 탐색"에서 "통일된 표준"으로
  국가 차원의 입법을 위한 실질적 기반 제공
```

### 6.2 수직적 심화: 낮은 민감도에서 높은 민감도로

```
영역 확장:

1단계: 공공 데이터 (정부 개방 데이터, 기상 데이터, 연구 데이터)
  → 가장 쉬움 — 정부는 데이터 개방 동기가 있음
  → 고품질, 논란 적음
  → 신뢰와 기술 기반 구축

2단계: 생활 데이터 (건강 데이터, 교통 데이터, 교육 데이터)
  → 사람들의 일상생활과 밀접하게 관련
  → 성공 시 가장 큰 성취감
  → 질병 검진 모델 하나가 생명을 구하는 것이 100편의 글보다 강력함

3단계: 경제 데이터 (소비 데이터, 공급망 데이터, 금융 데이터)
  → 핵심 이해관계에 접촉 — 이것이 진정한 "생산 수단"
  → 이 단계에서는 자본이 이미 반격을 시작함
  → 이제는 기술력이 아닌 정치력이 필요함

4단계: 알고리즘 관리권
  → 단순히 "너의 데이터는 너의 것"이 아니라 "너의 노동 과정은 너의 것"
  → 가장 철저한 변혁
```

## VII. 피해야 할 실수

### 7.1 역사적 교훈

**교훈 1: 1927년 도시 봉기의 실패**
- 상하이, 창사에서 노동자 무장 봉기 전개
- 모두 실패 — 도시에서 적의 통치가 너무 강력했음
- 교훈: **적이 가장 강한 곳에서 도전하는 것은 용기가 아니라 무모함이다**

**AI 시대에 적용:**
- 처음부터 구글, 오픈AI와 정면 승부를 시도하지 말라
- "AI 자본주의를 타도하려"고 하지 말라
- 적이 주목하지 않는 곳(공공 데이터, 건강 데이터, 농업 데이터)에 근거지를 건설하라

**교훈 2: 왕명(王明)의 "좌경" 모험주의**
- 홍군이 대도시를 공격하고 국민당 주력과 싸울 것을 요구
- 결과: 제5차 반포위 작전 실패, 장정(長征) 강요
- 교훈: **세력 균형이 불리할 때는 진지전을 하지 말고 유격전을 하라**

**교훈 3: 장국도(張國燾)의 분열주의**
- 장정 중 중앙 명령에 불복, 별도의 중앙위원회 설치
- 결과: 분열이 혁명 역량을 약화시킴
- 교훈: **전체 전략의 통일이 필요하다 — 분산 행동이 각자 제멋대로 하는 것은 아니다**

### 7.2 지금 피해야 할 실수

```
실수 1: 완벽한 계획을 기다림
  "이론이 충분히 완성되지 않았다, 모든 세부사항을 생각한 다음 행동하자"
  → 완벽한 계획은 결코 존재하지 않음
  → 실천이 진리를 판단하는 유일한 기준 — 먼저 시작하고, 실천 속에서 개선하라

실수 2: 성급한 확장
  "이 파일럯이 좋다, 전국적으로 즉시 확대하자"
  → 급할수록 느려진다 — 견고한 기반 없이 확장은 붕괴로 이어짐
  → 먼저 한 곳에서 깊이 뿌리내린 후 복제하라

실수 3: 폐쇄적 운영
  "우리가 설계하고, 구축하고, 운영하자"
  → 지역 대중과 통합되지 않는 조직은 실패할 수밖에 없음
  → 데이터 트러스트는 첫날부터 "국민 스스로의 프로젝트"여야 함

실수 4: 기술 결정론
  "좋은 기술 솔루션은 자연스럽게 사용자를 끌어들일 것이다"
  → 공공 트러스트의 핵심은 기술이 아님 (기술도 중요하지만)
  → 핵심은 신뢰, 거버넌스, 분배 — 이것들은 사회적 문제이지 기술적 문제가 아님

실수 5: 비관주의
  "이런 소규모가 무슨 소용인가? 큰 그림을 바꾸지 못할 것이다."
  → 별불지화(星火燎原), 하나의 불꽃이 대초원을 태울 수 있다
  → 성공적인 모든 파일럯은 "대안은 없다"는 주장에 대한 최고의 반박이다
```

## VIII. 결론: 징강산에서 전 세계로

1927년, 호남-강서 국경의 추수봉기가 실패했다. 모택동은 남은 부대를 이끌고 징강산으로 올라갔다.

그 누구도 그 당시 이 외딴 산악 근거지가 결국 중국 혁명 승리의 출발점이 될 것이라고는 상상하지 못했다.

AI 시대에도 마찬가지다. 오늘날 라틴아메리카의 어느 도시에 설립된 공공보건 데이터 트러스트는 — 소규모이고, 영향력은 제한적이며, 언제든지 폐쇄될 수 있지만 — 하나의 가능성을 대표한다: **AI 독점 자본을 넘어선 대안.**

> **"마르크스주의자는 점쟁이가 아니다. 그들은 미래 발전과 변화의 일반적 방향을 제시할 수 있을 뿐, 기계적으로 그 날과 시간을 규정할 수는 없다."**
> — 모택동, *「별불지화(星火燎原)」*

그러나 일반적 방향은 명확하다:

```
하나의 데이터 트러스트 → 열 개의 데이터 트러스트 → 백 개의 데이터 트러스트
하나의 도시 → 하나의 국가 → 하나의 대륙
공공보건 → 생활 데이터 → 경제 데이터
경제 실험 → 정치적 역량 → 체제 변혁
```

> **"온 나라가 마른 부싯깃으로 뒤덮여 있으며, 곧 불길을 뿜어낼 것이다."**

AI 시대도 마찬가지다 — 아니, **온 세계**가 마른 부싯깃으로 뒤덮여 있다. AI로 인한 대체 실업, 데이터 착취, 알고리즘 소외 — 이러한 모순들이 축적되고 있다. 우리에게 부족한 것은 "조건"이 아니라 단 하나의 **불꽃**이다.

> **"별불지화(星火燎原), 하나의 불꽃이 대초원을 태울 수 있다."**
