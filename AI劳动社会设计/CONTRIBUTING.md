# 贡献指南

## 欢迎

本项目是一个开放的思想工程，欢迎任何人的批评、讨论、改进。我们信奉：

> "真理越辩越明，智慧越分越增。"

## 贡献方式

### 1. 理论批评与补充

如果你发现理论盲区、逻辑漏洞、事实错误，欢迎指出。

```
示例：
- "马克思对数据所有制的分析忽略了交换环节……"
- "韦伯的形式理性/实质理性框架需要结合……"
- "建议补充某某理论家的视角……"
```

### 2. 制度设计的完善

如果某个制度设计在现实层面不可操作，或者你有更好的设计方案，欢迎提出。

### 3. 案例分析

如有关于AI劳动社会的真实案例（数据标注、算法管理、平台劳动等），欢迎提供。

### 4. 多语言翻译

多语言宣言目录在 `multilingual/` 目录下。如要添加新的语言翻译，请参照现有格式。

### 5. 技术实现

如果对技术白皮书（15号文档）中的技术架构有改进建议或实现经验，欢迎贡献。

## 贡献流程

### 小改动（错别字、小修正）

直接提交即可。

### 大改动（新增文档、重大修订）

1. 先阅读现有相关文档，理解已有框架
2. 建议先在讨论区提出你的想法，获得反馈后再撰写
3. 遵循现有的文档格式（参见现有文档的风格）
4. 更新术语表（08_术语表与跨理论索引.md）中的相关条目
5. 更新00_总目录与导航.md中的对应行
6. 更新CHANGELOG.md

## 文档规范

### 命名规范
- 文件名：`{编号}_{理论家/主题}_{核心内容}.md`
- 编号规则：01-20为理论诊断，21-40为制度设计，41-60为工具与传播
- 英文字母和阿拉伯数字均为半角

### 格式规范
- 标题使用 `#` 一级标题为文档标题
- 章节使用 `##` 到 `####`
- 表格用于概念对比和分类信息
- 代码块（```）用于结构示意和设计框架（非代码）
- 引用使用 `>` 用于经典原文引用
- 加粗 `**` 用于核心概念

### 内容规范
- 每个文档应该有"核心论题"
- 应该有"与其它文档的交叉引用"
- 应该有"延伸阅读"部分
- 结尾应该有"思考提问"

## 问与答

**Q: 我没有理论背景，可以贡献吗？**
A: 可以。实践经验和案例观察同样重要。

**Q: 我不同意文档中的观点怎么办？**
A: 欢迎提出不同意见。我们不是追求"统一答案"，而是追求更全面的理解。

**Q: 我可以把这个框架用在其他地方吗？**
A: 可以。CC BY-SA协议允许，但请遵守协议要求。
