## 一、核心的論題

> **「一国の内部で、四周を白色政権に包囲されながら、一小塊または若干小塊の赤色政権の区域が長期にわたって存在する——これは世界各国でかつてなかったことである。」**
> ——毛沢東『中国の赤色政権はなぜ存在しうるか』（1928年）

AI時代の変革も同じ構造的問題に直面している：**世界のAIインフラはほぼ100％が独占資本に支配されている——データは彼らのサーバーに、計算能力は彼らのチップに、モデルは彼らのパラメーターの中にある。このような条件で、データ公有制はどのようにゼロから始められるのか？**

答えは：**根拠地を建立することである。**

「旧制度を打倒してから新制度を築く」のではなく、旧制度の「隙間」にまず新制度の「拠点」を築く——新制度を局部的な範囲で稼働させ、その実現可能性を証明し、そして徐々に拡大するのである。

本稿の目標：データ公有制の**最初の根拠地**をどこに、どのように建設し、どのように存続させ、どのように拡大するかを設計すること。

## 二、なぜ根拠地が必要か？

### 2.1 根拠地なき方案はすべて空論

07号文書は「グローバルアルゴリズム条約」から始まる——これはトップダウンである。
09号文書は「国際的ガバナンス枠組み」から始まる——これは大から小へである。

両者に共通する問題：**第一歩をどう踏み出すかを述べていない。**

> かつて大革命初期、党内にも二つの傾向があった：
> - 一つは「都市中心論」——先に大都市を占領し、それから農村に拡大する（ソ連から学んだ）
> - もう一つは「全面開花」——全国各地で同時に蜂起し、直接旧政権を打倒する
> - 実践はこれらがいずれも誤りであることを証明した——正しい道は「農村が都市を包囲する」ことであった
>
> なぜか？農村に何か魔力があったからではない——理由は以下の通り：
> 1. 敵の力は大都市に最も集中していた——正面からぶつかっては勝てなかった
> 2. 敵の農村における支配は最も薄弱だった——生存空間があった
> 3. 農村人口が圧倒的多数を占めていた——大衆的基盤があった
> 4. 局部地域で政権を樹立できた——まず小範囲で新制度の実現可能性を証明できた

AI時代も同様である。

### 2.2 AI時代の「農村」はどこにあるのか？

「農村」は地理的概念ではなく、**構造的概念**である——すなわち、**敵の力が薄弱で、大衆の需要が切実であり、局部的に代替制度を樹立できる分野**を指す。

| 次元 | AI時代の「都市」（敵が強い） | AI時代の「農村」（突破可能） |
|------|--------------------------|--------------------------|
| 業種 | 検索エンジン、SNS、Eコマース、クラウドコンピューティング | 医療データ共有、農業データ協同、公共教育データ |
| 地域 | シリコンバレー、北京、深圳 | グローバル・サウスの二線都市、ヨーロッパの中小国 |
| データ | ユーザー行動データ（プラットフォームにロックイン済み） | 公共データ（政府データ、科研データ、気象データ） |
| 応用 | チャットボット、コンテンツ推薦 | 公共サービス（交通、環境、健康モニタリング） |

## 三、井岡山はどこにあるのか？

### 3.1 選択基準

私がかつて井岡山を選択した経験（羅霄山脉中段）によれば、良い根拠地は五つの条件を備えていなければならない：

```
① 敵の支配の薄弱環節
   → 独占資本が深く浸透していない分野
   → またはすでに代替案の生存空間が存在する分野

② 良好な大衆的基盤
   → 切実な社会的需要が存在し、現行の資本の方案では
   満たせない
   → 大衆が変革に参加する意欲を持っている

③ 持続可能な経済モデル
   → 自己維持・自己発展が可能
   → 外部からの資金援助に依存しない（または初期の限定的依存）

④ 「地形」の優位
   → 制度的保護空間（法の空白地帯、地方政府の支援）
   → 技術的な防御可能性（オープンソースアーキテクチャ、
   分散配置）

⑤ 拡張可能性
   → モデルが複製可能で普及可能
   → 一回限りの実験ではない
```

### 3.2 三つの候補根拠地

上記条件の総合評価に基づき、最も実行可能な三つの方向性：

#### 候補一：ラテンアメリカ諸国の公衆衛生データトラスト

**場所：** チリまたはブラジル

**なぜここか：**
- ラテンアメリカ諸国には強い「デジタル主権」の政治的意志がある——チリはデータ保護法を起草中、ブラジルはLGPD（一般データ保護法）を有する
- 公衆衛生は「低感度・高需要」分野である——「AIで病気を診断する」ことに反対する者はいない
- 中国/ファーウェイはラテンアメリカにインフラ支援の実績がある——海底ケーブル、データセンター（ただしファーウェイに依存してはならない——現地主導のプロジェクトであるべき）
- 人口規模が適切（チリ1900万人）で、試験的実施に適している

**具体的に何をするか：**
```
「国家公衆衛生データ公共トラスト」を設立：

データソース：
  → 公立病院の匿名化診療データ
  → 公衆衛生監視データ（感染症、慢性疾患）
  → 住民が自発的に提供する健康行動データ

データ用途：
  → 現地化されたAI診断モデルの訓練（ラテンアメリカの
  常見病——デング熱、シャーガス病、結核）
  → 公衆衛生リソース配分の最適化
  → 製薬研究開発（対価を支払わずにデータを使用する新薬は、
  その価格が公共トラストの制約を受ける）

ガバナンス構造：
  → 理事会：政府1/4 + 医療機関代表1/4 + 市民代表1/4 +
  技術専門家1/4
  → 収益分配：AIが生み出す経済効果（より精度の高い予防に
  よる医療費削減など）→ 50％は公衆衛生システムに還元、
  30％はデータ提供者（データ配当）、20％は運営コスト

成功指標：
  → 2年以内に公立病院データの50％以上をカバー
  → 少なくとも1つのAI診断モデルが実際に使用される
  （精度>90％）
  → 住民の自発的参加率>30％
  → データ公有制が私有制より効率的であることを証明——
  共有データで訓練したモデルの精度が単独病院のデータで
  訓練したモデルよりX％高い
```

**リスクと対応：**
| リスク | 対応 |
|--------|------|
| 米国の圧力（「データ公有化は没収だ」） | これは「公共トラスト」であって「没収」ではないと強調——公共図書館と同様 |
| 政権交代による政策中断 | 行政命令ではなく法律に明記；超党派の支持を獲得 |
| 技術能力の不足 | 現地大学と協力；オープンソースAIツールを活用 |
| データ品質の低さ | 段階的に——まず高品質の大病院データから始め、その後基礎医療機関をカバー |

#### 候補二：ヨーロッパ都市の交通データ協同組合

**場所：** ドイツのある都市（ハンブルクなど）またはオランダのある都市（ユトレヒトなど）

**なぜここか：**
- EUは世界最強のデータ保護法制（GDPR）を有し、現在「データガバナンス法」（DGA）を制定中——法的環境が最も友好的
- 欧州の都市には強力な市政運営の伝統がある——交通は市政府が直接管理するサービス
- 欧州には協同組合の法的枠組みがある——データ協同組合に明確な法的地位が存在

**具体的に何をするか：**
```
「都市交通データ協同組合」を設立：

データソース：
  → 市民が自発的に提供するGPS移動データ（アプリ経由）
  → 公共交通システムの運行データ
  → 市政センサーの交通流量データ

データ用途：
  → 公共交通の路線と運行頻度の最適化
  → 渋滞の緩和（AIによる信号機制御）
  → 都市計画の意思決定への根拠提供

ガバナンス構造：
  → 協同組合員（データ提供者）は一人一票
  → 管理委員会を選出
  → データを使用する外部組織（営利企業）は対価を支払い、
  収益は協同組合員に帰属

資本主義的方案との違い：
  → Uber/Wazeの方案：ユーザーデータ収集→ナビ最適化→
  広告販売/データ販売→株主への利益配分
  → 協同組合方案：市民データ収集→交通最適化→公共交通改善
  →市民への還元
  → 技術はほぼ同じだが、所有権と分配が全く異なる
```

**注意：** 欧州は試験的実施に適しているが、欧州の「改良主義」的伝統は協同組合の段階で止まらせる可能性がある——所有権そのものに挑戦することなく。ラテンアメリカでの試験的実施の方が変革の可能性が大きい——ラテンアメリカの不平等はより極端であり、変革の駆動力がより強いからである。

#### 候補三：中国のある都市の「配達員データ協同組合」

**場所：** 中国のある二線都市（長沙、成都など）

**なぜここか：**
- 中国は世界最大規模のギグワーカー集団を有する——美団（Meituan）だけでも700万人以上の配達員
- 中国のデジタルインフラは最も発達している——しかしデータはすべてプラットフォームに帰属
- 配達員の搾取度は世界最高——プラットフォームの取り分は20-30％、労働保障なし
- 中国には労働組合の法的枠組みがある——現在独立した労働組合はないが、「団体交渉」は法律上認められている

**具体的に何をするか：**
```
「配達員データ協同組合」を設立（最も簡単なことから始める）：

第一歩：共同監視
  → 配達員が共同でプラットフォームのアルゴリズムルールの
  変化を記録・分析
  → 「アルゴリズムルール追跡データベース」を構築——
  プラットフォームがルールを変えたら、全員で一緒に分析
  → これは「技術的対抗」ではなく、「情報の対称化」である

第二歩：団体交渉
  → 配達員がプラットフォームのアルゴリズムルールのデータを
  持てば、交渉力が生まれる
  → 「単価がX以下になったら、全員15分間オフライン」
  → データが集団行動を可能にする

第三歩：自主プラットフォーム
  → 配達員協同組合が独自の配車アプリをリリース
  → 初期は「非ピーク時」から始められる——プラットフォームが
  引き受けたがらない注文（深夜配達、長距離配達）
  → 徐々に拡大
  → 協同組合は手数料を取らないか、5％（運営コスト用）とし、
  残りはすべて配達員に還元

成功指標：
  → 12ヶ月以内に1000人以上の配達員をカバー
  → 協同組合プラットフォームの配達効率が商業プラットフォーム
  と同等以上
  → 配達員の収入が20％以上向上（手数料引き下げのみで）
  → データ所有権が真に配達員の手にある
```

**リスク警告：**
- 中国の政治環境は独立組織に対して最も厳しい——これは高リスクの選択肢である
- しかし中国での成功のデモンストレーション効果も最大である——中国はデジタルギグエコノミーの世界的中心だから
- **戦略：現行の法的枠組みに直接対抗せず、「労働者権益」と「データ安全法」の枠組み内で運営する**

---

### 3.3 私の推薦

総合評価の結果、**最初の根拠地は最もラテンアメリカ（候補一）に建設されるべきである**。

理由：
1. 政治的空间が最大——ラテンアメリカ諸国はデジタル主権の政治的意志を持つ
2. 国際的影響力が強い——成功すれば他のグローバル・サウス諸国を牽引する
3. リスクが管理可能——米国や中国の核心的利益に直接対抗しない
4. 利用可能なインフラがある——ファーウェイなど中国企業がラテンアメリカに協力基盤を持つ
5. デモンストレーション効果が良い——ラテンアメリカからアフリカ、アジアへ再現可能

## 四、根拠地の経済モデル

根拠地が生き残れるかどうかは、**自分で自分を養えるかどうか**に鍵がある。

### 4.1 収入源

データ公共トラストは経済的に自立していなければならない——外部からの資金援助なしで運営できなければならない。

```
初期（1-2年）：
  スタートアップ資金：公的資金（政府助成金、研究助成金）
  または公益基金から
  規模が小さく、コストも低い——一都市のデータトラスト、
  サーバー＋人員、年間運営コスト約50-200万米ドル

中期（2-5年）：
  主な収入：データ使用許可料
  保険会社（健康データで保険商品を最適化）→ 支払い
  製薬会社（医学データで新薬研究開発を支援）→ 支払い
  都市計画部門（交通データで都市管理を最適化）→ 支払い
  （政府内部予算）
  → 商業ユーザーからの課金で運営コストを十分にカバー

成熟期（5年以上）：
  データ配当の分配——運営コストをカバーするほかに、
  余剰をデータ提供者に分配
  これは「データ公有制は持続可能である」ことを証明する
```

### 4.2 分配メカニズム

```
収入分配式：

年間総収入 = Σ(商業ユーザーデータ使用許可料) + 公共予算支援

分配優先順位：
  第一：運営コスト（サーバー、帯域、人件費）——約30％
  第二：公共準備基金（技術向上、リスク対応）——約20％
  第三：データ配当（データ提供者に分配）——約50％

個人データ配当の分配：
  基本分（60％）：全データ提供者に均等に分配
  ——データの本質は社会的なものであり、個人的なもの
  ではないから
  ——一人ひとりのデータの個別的価値は限定的だが、
  集団的価値は莫大である
  貢献分（40％）：データ量・品質に応じて変動
  ——必要なインセンティブメカニズム——そうでなければ
  誰もより多くのデータを提供しようとしない
```

## 五、根拠地の組織モデル

### 5.1 ガバナンス原則

```
データ公有制における民主集中制の応用：

民主：
  → データ提供者（すなわち「データ市民」）が一人一票で
  理事会を選出
  → 重要決定（データ許可、収益分配比率、ルール変更）は
  必ず全体投票を経る
  → アルゴリズムの透明性——すべてのデータ使用記録が
  組合員に可視化される

集中：
  → 理事会が日常管理の決定を担当
  → 技術チームに専門的自主権がある（「すべてのコード行に
  投票が必要」ではない）
  → 一度決定が下されたら、実行しなければならない——
  これが効率の保障である

鍵：民主と集中のバランス
  → 民主なき集中 = 新たな形の「データ官僚主義」
  → 集中なき民主 = 非効率、何の決定もできない
```

### 5.2 組織構造

```
データ公共トラスト組織構造図：

┌──────────────────────────────────────────────┐
│              データ市民大会                      │
│          （すべてのデータ提供者、一人一票）          │
│            最高権力機関                        │
├──────────────────────────────────────────────┤
│                   選挙                         │
│                    ▼                          │
├──────────────────────────────────────────────┤
│              理事会（7-11人）                  │
│    データ提供者代表4人 + 技術専門家2人            │
│    + コミュニティ代表2人 + 政府代表1人            │
│             （拒否権なし）                      │
│    任期2年、再任1回限り                        │
├──────────────────────────────────────────────┤
│       任命        │        選挙                │
│        ▼          │         ▼                 │
│  実行チーム        │  アルゴリズム監視委員会        │
│  （日常運営）       │  （監査、苦情、申し立て）       │
└──────────────────────────────────────────────┘
```

## 六、根拠地から全国へ——拡大の道筋

### 6.1 水平的拡大：1から10へ

一つのデータトラスト試験が2-3年間成功裏に運営されたら、複製を開始する：

```
複製戦略：

第一歩：ドキュメント化——試験のすべてのプロセス、
技術方案、法律文書を「オープンソースツールキット」にする
  次の都市はゼロから始める必要がない
  複製コストを低減する

第二歩：トレーニング——最初の試験の運営者が「シード
教官」となる
  次の都市に行き、新しい試験の設立を支援する
  経験が最高の教材である

第三歩：ネットワーク化——各試験の間にデータ共有協定
を確立する
  データトラスト同士が相互接続——データがより広範囲で
  流れ、価値が大きくなる
  「データトラスト連盟」を形成——共同交渉、共通基準、
  共通ガバナンス

第四歩：標準化——連盟を基盤に「データトラスト国家基準」
を策定する
  「各自の模索」から「統一規範」へ
  国家レベルの立法に実践的根拠を提供する
```

### 6.2 垂直的深化：低感度から高感度へ

```
拡大分野：

第一段階：公共データ（政府開放データ、気象データ、
科研データ）
  → 最も容易——政府にデータ開放の動機がある
  → データ品質が高く、論争が少ない
  → 信頼と技術的基盤を確立する

第二段階：民生データ（健康データ、交通データ、
教育データ）
  → 民衆の日常生活と密接に関連
  → 成功の実感が最も強い
  → 一つの疾病早期スクリーニングモデルが人命を救う →
  百の記事よりも効果的

第三段階：経済データ（消費データ、サプライチェーン
データ、金融データ）
  → 核心的利益に触れる——これこそ真の「生産手段」
  → この段階になると、資本が反撃を開始する
  → この時は技術力ではなく政治力で推進する必要がある

第四段階：アルゴリズム管理権
  → データの公有だけでなく、「アルゴリズム権力」の
  民主化
  → 「あなたのデータをあなたが管理する」から「あなたの
  労働プロセスをあなたが管理する」へ
  → これが最も徹底した変革である
```

### 6.3 経済から政治へ

根拠地は単なる「経済実験」ではなく、同時に「政治的主体」でもある。

```
データトラストの政治的意義：

1. 新しい社会関係のミクロモデルである
  → ここではデータは商品ではなく、公共の富である
  → ここではアルゴリズムはブラックボックスではなく、
  透明なツールである
  → ここでは労働者は道具ではなく、意思決定者である
  → 「もう一つのAI社会」が実現可能であることを示す

2. 幹部を鍛える学校である
  → データトラストの運営には技術、ガバナンス、法律、
  組織能力が必要
  → これらの能力は実践によってのみ培われる
  → 成功した運営のたびに、未来の変革リーダーが育成される

3. 力の蓄積点である
  → データトラストはデータを管理するだけでなく、
  「人」も管理する
  → データ提供者間のつながり → 政治組織に発展する可能性
  → データトラストは「労働組合＋協同組合＋公共事業」の
  ハイブリッドでありうる
```

## 七、根拠地が避けるべき誤り

### 7.1 歴史的教訓

**教訓一：1927年の都市蜂起の失敗**
- 上海、長沙などの大都市で労働者武装蜂起を発動
- すべて失敗——敵の都市における支配が強すぎたから
- 教訓：**敵の力が最も強い場所で挑戦するのは勇敢さではなく、無謀である**

**AI時代への対応：**
- 最初からGoogleやOpenAIと正面から競争しようとするな
- 「AI資本主義を打倒する」ことを試みるな
- 敵が重視しない場所（公共データ、衛生データ、農業データ）に根拠地を築け

**教訓二：王明の「左」傾冒険主義**
- 紅軍に大都市を攻撃させ、国民党主力と決戦させるよう要求
- 結果：第五次反包囲殲滅作戦が失敗し、長征を余儀なくされる
- 教訓：**力量比較が不利な時は、陣地戦をしてはならない——遊撃戦を戦え**

**AI時代への対応：**
- 「一つの方案でAI業界全体を変えられる」と期待するな
- 短期的な制度変革の可能性を過大評価するな
- 「遊撃戦」を戦え——局部地域で小さな勝利を収め、力を蓄積せよ

**教訓三：張国燾の分裂主義**
- 長征の途中、中央の指揮に従わず、別の中央を樹立
- 結果：分裂が革命力を弱体化させた
- 教訓：**全体戦略の統一は必要である——分散行動は各々が勝手にやることとは違う**

**AI時代への対応：**
- 各根拠地のデータトラストは共通の原則と基準に従うべき
- 「各自が独自のデータ公有制を発明する」のではない
- 統一された理論的枠組み（このリポジトリが提供する）と実践ガイドを持つべき

### 7.2 現在避けるべき誤り

```
誤り一：完璧な方案を待つ
  「理論がまだ十分に完成していない。細部がすべて
  明確になってから動こう」
  → 完璧な方案など永遠にない
  → 実践は真理を検証する唯一の基準である——
  まずやり始め、やりながら改善せよ

誤り二：急いで拡大する
  「この試験は良い、すぐに全国展開しよう」
  → 急いては事をし損じる——安定した基盤なくして
  拡大すれば崩壊するだけ
  → まず一箇所で深く根を張り、それから複製せよ

誤り三：閉門造車（世間に背を向けて独りよがり）
  「我々で設計し、建設し、運営する」
  → 現地の大衆と結合しない組織は必ず失敗する
  → データトラストは最初から「大衆自身のプロジェクト」
  でなければならない

誤り四：技術決定論
  「技術方案が良ければ、自然に人が来る」
  → 公共トラストの核心は技術ではない（技術も重要だが）
  → 核心は信頼、ガバナンス、分配である——これらは
  社会問題であり、技術問題ではない

誤り五：悲観主義
  「この程度の規模で何になる？大局は変えられない。」
  → 星火燎原（一つの火花が野原を焼き尽くす）
  → 成功した試験のひとつひとつが「選択肢などない」
  という論への最良の反論である
```

## 八、結論：井岡山から全中国へ

1927年、湘贛辺境の秋収起义は失敗した。毛沢東は残存部隊を率いて井岡山に上った。

当時、誰もこの辺境の山岳根拠地が、やがて中国革命勝利の出発点になるとは想像できなかった。

AI時代も同様である。今日、ラテンアメリカのある都市で設立される公衆衛生データトラスト——規模は小さく、影響は限定的で、いつ閉鎖されてもおかしくない——しかしそれは一つの可能性を表している：**AI独占資本以外のもう一つの選択肢である。**

> **「マルクス主義者は易者ではない。未来の発展と変化について、ただ大きな方向を語るべきであり、機械的に期日を定めるべきではないし、また定めることもできない。」**
> ——毛沢東『星火燎原』

しかし大きな方向は明らかである：

```
一つのデータトラスト → 十のデータトラスト → 百のデータトラスト
一つの都市 → 一つの国家 → 一つの大陸
公衆衛生 → 民生データ → 経済データ
経済実験 → 政治力 → 制度変革
```

> **「中国はいたるところに乾いた薪が敷き詰められており、まもなく烈火のごとく燃え上がるであろう。」**
>
> AI時代も同じである——いや、全国ではなく——全世界——いたるところに乾いた薪が敷き詰められている。
> AI代替失業、データ搾取、アルゴリズム疎外——これらの矛盾が蓄積している。
> 我々に欠けているのは「条件」ではなく、一点の**星火**である。

> **「私の言う中国革命の高まりがまもなく到来するというのは、ある人々が言うような『到来の可能性がある』というような、全く行動上の意味のない、手の届かない空虚なものでは決してない。それは海岸に立って遠く海の中に帆先の尖りをすでに見ることのできる一隻の航船であり、高山の頂に立って東の方に光芒四射・噴薄欲出の一輪の朝日を遠望するものであり、母胎の中で躁動する、まもなく成熟しようとしている一人の嬰児である。」**
>
> AI時代の新社会は、旧社会の母胎の中で躁動している。
