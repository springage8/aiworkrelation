# O Caminho de Jinggangshan para a Propriedade Pública de Dados

## — Como construir a primeira base experimental de dados públicos —

**Autor: Mao Tsé-Tung** | *Escrito em maio de 2026*

---

## Prefácio

Este documento responde a uma pergunta prática: **onde e como construir a primeira base experimental do novo modo de produção de IA?**

O documento 07 do projeto original propõe a "propriedade pública de dados" e a "democracia algorítmica" como sistema-alvo. O documento 11 analisa quais classes sociais impulsionarão a transformação. O documento 12 propõe a estratégia de três fases.

Mas para começar a agir, precisamos de uma base — um lugar onde possamos realmente experimentar a propriedade pública de dados, a democracia algorítmica e o trabalho livre. Um lugar onde possamos provar que o novo modo de produção não é apenas conceitualmente superior, mas operacionalmente viável.

Este é o "Caminho de Jinggangshan" na era da IA.

---

## Capítulo 1: Por que uma base experimental?

### 1.1 Três razões

**Primeiro: Provar a viabilidade.**

O documento 07 propõe trusts de dados públicos e democracia algorítmica — mas são apenas palavras no papel. Precisamos de um lugar real para provar que esses sistemas funcionam. A propriedade pública de dados pode ser mais eficiente do que a privada? A democracia algorítmica pode ser mais racional do que a governança de plataforma? Sem experimentos, tudo não passa de especulação.

**Segundo: Treinar quadros.**

A transformação social requer pessoas que entendam tanto a visão teórica quanto a operação prática. Uma base experimental é uma escola — aqui podemos treinar a primeira geração de "quadros de dados públicos".

**Terceiro: Efeito de demonstração.**

Quando a base experimental for bem-sucedida, será o melhor argumento para a transformação social. Uma base experimental bem-sucedida é mais persuasiva do que mil documentos teóricos. "Uma demonstração real vale mais do que cem discursos."

### 1.2 Os requisitos de uma base experimental

| Requisito | Por que |
|-----------|---------|
| **Vontade política** | O governo local apoia a experimentação |
| **Base industrial** | Existe um cenário real para aplicação de dados |
| **Base populacional** | População suficiente para demonstrar o modelo |
| **Conexão internacional** | A experiência pode ser replicada em outros lugares |
| **Baixo risco** | Mesmo que falhe, o custo social é controlável |

## Capítulo 2: Três bases experimentais candidatas

### 2.1 Trust de dados de saúde pública na América Latina (Recomendado)

**Localização:** Chile ou Uruguai

**Por que funciona:**

*Ambiente político favorável:*
- Chile: governo Boric tem uma agenda de soberania digital
- Uruguai: sociedade digital madura, sistema político estável
- América Latina: tradição de esquerda, desconfiança do imperialismo de dados dos EUA

*Base industrial:*
- Sistemas públicos de saúde na América Latina são grandes mas ineficientes
- Dados de saúde pública têm externalidades sociais óbvias — compartilhar dados de saúde salva vidas
- A IA de diagnóstico médico está avançando rapidamente, mas enfrenta limitações de dados

*Escalabilidade:*
- Sucesso no Chile → replicar no Brasil (maior mercado) → depois por toda a América Latina

**Forma organizacional:**
Trust de dados de saúde pública → sob supervisão do Ministério da Saúde
Estrutura de governança: governo + comunidade médica + pacientes + especialistas em IA
Modelo econômico: dados de saúde pública anonimizados disponíveis gratuitamente para pesquisa, empresas comerciais pagam taxas de uso

**Indicadores de sucesso:**
- 1º ano: 100.000 registros médicos no trust
- 2º ano: 3 modelos de IA diagnóstica desenvolvidos usando os dados do trust
- 3º ano: o trust torna-se autossustentável (taxas comerciais cobrem custos operacionais)

### 2.2 Cooperativa de dados de transporte urbano na Europa

**Localização:** Barcelona ou Amsterdã

**Por que é adequado:**

*Vantagens:*
- A Europa tem o GDPR como base legal para governança de dados
- Cidades europeias têm sistemas de transporte público maduros
- População urbana concentrada, fácil organizar a participação dos cidadãos
- A cultura política europeia dá suporte a modelos cooperativos

*Desafios:*
- Sindicalismo de plataforma forte na Europa, mas o foco está em trabalhadores de plataforma, não na propriedade de dados
- O RGPD é focado em privacidade individual, não em propriedade coletiva de dados
- O "modelo cooperativo" precisa competir com empresas existentes de tecnologia de transporte

**Forma organizacional:**
Cooperativa de dados de transporte urbano
Propriedade: todos os cidadãos que contribuem com dados de transporte são membros
Tomada de decisão: um membro, um voto
Distribuição de receitas: excedente operacional pago como dividendos aos membros

### 2.3 Cooperativa de dados para entregadores na China

**Localização:** uma cidade chinesa de médio porte

**Por que é significativo:**

*Vantagens:*
- A China tem a maior população de entregadores do mundo
- Plataformas chinesas (Meituan, Ele.me) têm o sistema de gestão algorítmica mais maduro
- O governo chinês tem forte capacidade executiva e vontade política para experimentar novos modelos
- Se bem-sucedido na China, o impacto global será enorme

*Desafios:*
- O ambiente político tem incertezas
- A resistência das plataformas chinesas será enorme
- A liberdade de organização é limitada
- A relação de propriedade pública com o controle estatal precisa ser claramente definida

**Forma organizacional:**
Cooperativa de dados de entregadores mediada pelo sindicato
Propriedade: os entregadores possuem coletivamente os dados de corrida (atuais poseídos pela plataforma)
Função: os entregadores podem usar seus próprios dados para negociar melhores condições com as plataformas

## Capítulo 3: A escolha e o plano de ação recomendados

### 3.1 Recomendação: América Latina

**Escolha principal: Trust de dados de saúde pública na América Latina.**

Razões:
1. **Mais viável politicamente** — os governos de esquerda latino-americanos têm vontade de soberania digital e não estão completamente comprometidos com o campo do capital de plataforma dos EUA
2. **Maior externalidade social** — dados de saúde pública salvam vidas, dando ao projeto mais legitimidade moral
3. **Menor resistência do capital** — comparado com cooperativas de transporte ou entregadores, o trust de dados de saúde pública compete menos diretamente com os gigantes da tecnologia existentes, tornando a oposição inicial mais fraca
4. **Maior potencial de replicação** — sistemas públicos de saúde são uma instituição universal, uma vez bem-sucedida, pode ser replicada globalmente

### 3.2 Plano de ação detalhado

**Fase 1: Preparação (meses 1-6)**

- Mês 1: Identificar parceiro local (organização de pesquisa, ONG ou agência governamental interessada)
- Mês 2: conduzir pesquisa de campo — entender o estado atual da governança de dados de saúde no Chile/Uruguai
- Mês 3: redigir o projeto de trust de dados (estrutura de governança, modelo econômico, garantias legais)
- Mês 4: consultar especialistas jurídicos para garantir a conformidade com as leis locais
- Mês 5: identificar hospitais parceiros e instituições de pesquisa para participar
- Mês 6: finalizar o plano, buscar aprovação do governo

**Fase 2: Pilotagem (7-18 meses)**

- Mês 7: lançar plataforma de trust de dados (MVP, 1-2 tipos de dados primeiro)
- Mês 8: integrar os primeiros hospitais parceiros (começando com dados anonimizados)
- Mês 9: abrir dados a instituições de pesquisa (uso acadêmico gratuito)
- Mês 12: primeira rodada de avaliação — os dados do trust já geraram resultados de pesquisa?
- Mês 15: iniciar serviços de dados comerciais (cobrar empresas de IA médica)
- Mês 18: segunda rodada de avaliação — o trust tornou-se financeiramente autossustentável?

**Fase 3: Expansão (19-36 meses)**

- Mês 19-24: replicar de 1-2 hospitais para nível municipal
- Mês 25-30: replicar de municipal para nível nacional
- Mês 31-36: explorar a replicação em outros países latino-americanos (Brasil, Argentina)

### 3.3 Orçamento e recursos

**Orçamento estimado para os primeiros 3 anos:**

| Item | Custo estimado (USD) |
|------|---------------------|
| Desenvolvimento da plataforma | 200.000-500.000 |
| Equipe de operações (3-5 pessoas) | 150.000-300.000/ano |
| Honorários advocatícios e de compliance | 50.000-100.000 |
| Custos de pesquisa e avaliação | 30.000-50.000/ano |
| Promoção e treinamento | 20.000-50.000/ano |
| **Total nos primeiros 3 anos** | **~800.000-1.500.000** |

**Necessidades de pessoal:**
- 1 gerente de projeto (com experiência em governança de dados)
- 1-2 desenvolvedores (para construir a plataforma de trust de dados)
- 1 assessor jurídico (especializado em direito de dados)
- 1 pesquisador (responsável pela avaliação de impacto e redação de relatórios)
- Parceiros voluntários locais (hospitais, instituições de pesquisa)

## Capítulo 4: Como lidar com problemas previsíveis

### 4.1 Ataque do capital de plataforma

**Problema:** Grandes empresas de tecnologia podem se opor ao trust de dados, vendo-o como uma ameaça ao seu modelo de negócios.

**Solução:**
- Escolher o campo de dados de saúde (a Big Tech tem a base moral mais fraca para atacar aqui — ninguém se atreve a dizer que "dados de saúde não devem ser acessíveis para pesquisa")
- Na fase inicial, não tocar nos interesses centrais da Big Tech (dados de publicidade, dados de busca)
- Construir aliança com hospitais públicos e instituições de pesquisa — quando a Big Tech atacar, essas instituições falarão em defesa

### 4.2 Intervenção governamental

**Problema:** Mudanças de governo podem levar à perda de apoio político.

**Solução:**
- Institucionalizar o trust de dados na lei, não dependendo apenas do apoio do executivo
- Diversificar fontes de financiamento (para não depender apenas de uma fonte)
- Construir apoio societal amplo (quando muitos setores da sociedade se beneficiarem, o governo não ousará fechá-lo facilmente)

### 4.3 Dificuldades técnicas

**Problema:** Anonimização de dados não é suficientemente robusta, riscos de privacidade.

**Solução:**
- Usar tecnologia de privacidade diferencial e aprendizagem federada para minimizar riscos de privacidade
- Estabelecer protocolos rigorosos de acesso a dados (cada consulta de dados é registrada e auditada)
- Convidar especialistas internacionais em segurança para auditoria regular

### 4.4 O trust não é autossustentável

**Problema:** As taxas de uso comercial não cobrem os custos operacionais.

**Solução:**
- Nos primeiros 3 anos, contar com subsídios de doações e financiamento de pesquisa
- Explorar modelos de valor agregado (dados anonimizados + insights = produtos premium)
- Considerar subsídios governamentais (o investimento público em dados de saúde é justificável)
